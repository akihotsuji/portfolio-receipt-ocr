# 領収書 OCR - 要件定義書

## 1. プロジェクト概要

### 1.1 目的

個人事業主が確定申告に利用する領収書を OCR で読み取り、データ化・エクスポートするマイクロアプリ。
個人開発の学習プロジェクトとして、以下のテーマを実践的に習得することを主目的とする。

### 1.2 学習テーマ

| テーマ               | 概要                                                          |
| -------------------- | ------------------------------------------------------------- |
| ファイルアップロード | オブジェクトストレージ（S3互換）への保存と署名付き URL        |
| 非同期処理           | キューイング、ワーカー、ステータス管理                        |
| 並列処理             | OCR N並列 / エクスポート M並列の同時実行制御                  |
| 失敗系設計           | 自動リトライ、タイムアウト、冪等性の保証                      |
| AWS Lambda           | SQS トリガーによるサーバーレス関数の実装                      |
| MyBatis              | ORM として MyBatis を採用し、SQL ベースのデータアクセスを習得 |
| React Context        | 外部ライブラリに頼らないグローバル状態管理を習得              |
| Jest                 | フロントエンドのテスティングを習得                            |

### 1.3 技術スタック

| レイヤー       | 技術                                          |
| -------------- | --------------------------------------------- |
| バックエンド   | Spring Boot 4.x + Java 21 + PostgreSQL 16     |
| ORM            | MyBatis（SQL マッパー）                       |
| サーバーレス   | AWS Lambda（Java 21）                         |
| IaC            | AWS SAM                                       |
| フロントエンド | React 19 + TypeScript + Vite + Tailwind CSS   |
| 状態管理       | React Context（外部ライブラリ不使用）         |
| テスト（FE）   | Vitest                                        |
| OCR エンジン   | Tesseract OCR（Tess4J 経由）                  |
| ストレージ     | MinIO（ローカル） / S3（本番）                |
| 認証           | JWT 検証（Portal API の JWKS から公開鍵取得） |

## 2. 機能要件

### 2.1 OCR 読み取り項目

領収書から以下の5項目を読み取る。確定申告の帳簿記載に必要な情報がベース。

| #   | 項目           | 優先度 | 備考          |
| --- | -------------- | ------ | ------------- |
| 1   | 日付           | 必須   | 取引日        |
| 2   | 金額（合計）   | 必須   | 支払総額      |
| 3   | 支払先（店名） | 必須   | 取引先名      |
| 4   | 品目（明細）   | あれば | 何を買ったか  |
| 5   | 税区分         | あれば | 8% / 10% など |

精度は完璧でなくてよい。「それなりに読める」レベルで十分とする。

### 2.2 ユーザーフロー

2つの独立したフローで構成する。

#### フロー①: OCR 読み取り → 確認・編集 → 確定

```
ファイル選択 → アップロード → OCR 処理（非同期）→ 結果表示 → 編集 → 確定・保存
```

- 1回のアップロードで最大10枚まで選択可能
- サーバー側で非同期・並列に OCR 処理
- 各画像ごとに処理状況（処理中 / 完了 / 失敗）を表示
- 完了した画像は、プレビューと読み取り結果の編集フォームを並べて表示
- ユーザーが確認・修正して「確定」するとサーバーに保存
- 画面B の左サイドバーに過去のバッチ履歴一覧を表示し、過去のバッチに随時アクセス可能

#### フロー②: データ一覧 → 選択 → エクスポート

```
一覧表示 → ソート・フィルタ → 対象選択 → CSV エクスポート
```

- 確定済みの読み取りデータを一覧表示
- 日時・品目などでソート・フィルタ可能
- チェックボックスでエクスポート対象を選択し CSV 出力

### 2.3 画面構成

| 画面 | 名称                          | 主な機能                                                               |
| ---- | ----------------------------- | ---------------------------------------------------------------------- |
| A    | アップロード画面              | ドラッグ＆ドロップ or ボタンでファイル選択、アップロード開始           |
| B    | 処理状況 + 確認・編集画面     | バッチ履歴一覧（サイドバー）、ステータス一覧、画像プレビュー、読み取り結果の編集フォーム、確定ボタン |
| C    | データ一覧 + エクスポート画面 | 確定済みデータ一覧、ソート・フィルタ、エクスポート実行                 |

## 3. 非機能要件

### 3.1 ファイルアップロード

| 項目                   | 仕様                                                      |
| ---------------------- | --------------------------------------------------------- |
| 対応形式               | JPEG, PNG, PDF                                            |
| 最大ファイルサイズ     | 10MB / 枚                                                 |
| 最大同時アップロード数 | 10枚                                                      |
| 保存先                 | MinIO（S3互換）の専用バケット `receipt-uploads`（非公開） |
| アクセス制御           | 署名付き URL による限定公開                               |

- PDF: ダウンロードした電子領収書を想定
- JPEG / PNG: 店舗レシートをスマホで撮影した写真を想定
- 既存の `frontend-assets` バケットとは分離して管理

### 3.2 非同期処理・並列処理

| 項目                   | 仕様  |
| ---------------------- | ----- |
| OCR 同時処理数         | N = 3 |
| エクスポート同時処理数 | M = 3 |

- 10枚アップロードされても同時に処理するのは3枚まで、残りはキューで待機
- キューの滞留や処理状態の遷移が目に見えることで学習効果を高める
- 並列数の上限設定自体にはコストなし（リクエスト時のみリソース消費）

### 3.3 失敗系設計

| 項目         | 仕様                   |
| ------------ | ---------------------- |
| 自動リトライ | 最大3回                |
| タイムアウト | 60秒（画像・PDF 共通） |

- タイムアウト超過はリトライ対象として扱う
- 3回リトライしても失敗した場合:
  - 画面B に「失敗」ステータスを表示
  - ユーザーが手動で再試行ボタンを押して再実行可能
- 冪等性を保証し、同一ファイルの重複処理を防止する

## 4. 外部依存

### 4.1 OCR エンジン: Tesseract OCR

**採用理由:**

- 無料（Apache 2.0 ライセンス）
- ローカル実行のため外部 API への依存なし
- Docker コンテナに含めることで、ローカル開発環境と AWS 本番環境で同一イメージが動作
- Tess4J（Java ラッパー）により Spring Boot と直接統合可能
- GPU 不要で軽量に動作
- 日本語の学習済みデータ（`jpn.traineddata`）が提供されており、追加学習は不要

**不採用としたエンジン:**

| エンジン                | 不採用の理由                                                                                                              |
| ----------------------- | ------------------------------------------------------------------------------------------------------------------------- |
| Google Cloud Vision API | 精度は高いが従量課金が発生し、学習目的にはオーバースペック                                                                |
| AWS Textract            | 同上                                                                                                                      |
| NDLOCR-Lite             | 図書・雑誌向けに最適化されており領収書用途とはユースケースが異なる。Java 連携も外部プロセス呼び出しとなり統合コストが高い |

### 4.2 PDF 変換

- PDF → 画像変換には Apache PDFBox 等を使用
- 1ページ目のみを対象とする

## 5. スコープ外（やらないことリスト）

| スコープ外                 | 理由                                                                                     |
| -------------------------- | ---------------------------------------------------------------------------------------- |
| 多言語対応                 | 日本語の領収書のみ対象                                                                   |
| モバイル専用 UI            | レスポンシブは最低限対応するが、ネイティブアプリは作らない                               |
| OCR 精度の追求             | Tesseract の標準精度で十分。チューニング・追加学習はしない                               |
| インボイス制度への完全対応 | 適格請求書番号の読み取り等は対象外。個人情報の取り扱いが重くなるため学習スコープから除外 |
| 会計ソフト連携             | CSV 出力までがスコープ。freee 等への API 連携はしない                                    |
| 複数ページ PDF             | 1ページ目のみ対象。複数ページ対応は処理が複雑になるため除外                              |

## 6. 処理フロー概要図

```
[フロントエンド]
    │
    │  ファイルアップロード（最大10枚）
    ▼
[Spring Boot API]
    │
    ├─→ MinIO (receipt-uploads) にファイル保存
    │
    ├─→ キューにOCRジョブを投入
    │
    ▼
[OCR ワーカー] × 3並列
    │
    ├─→ MinIO からファイル取得
    ├─→ (PDF の場合) PDFBox で画像変換
    ├─→ Tesseract OCR で読み取り
    ├─→ 結果を DB (PostgreSQL) に保存
    │
    │  失敗時: 自動リトライ（最大3回 / タイムアウト60秒）
    │
    ▼
[フロントエンド]
    │
    ├─→ 画面B: ステータス確認 → 結果編集 → 確定
    │
    ├─→ 画面C: 一覧表示 → CSV エクスポート
    │         (AWS Lambda で CSV 生成)
    ▼
```
