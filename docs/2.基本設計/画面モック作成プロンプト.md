# 画面モック作成プロンプト

以下の仕様に基づいて、3つの画面のUIモック（ワイヤーフレーム）を作成してください。

---

## プロジェクト概要

**アプリ名**: 領収書 OCR
**目的**: 個人事業主が確定申告用の領収書をOCRで読み取り、データ化・CSVエクスポートするWebアプリ
**技術スタック（フロントエンド）**: React 19 + TypeScript + Tailwind CSS
**デザイントーン**: 業務アプリ。おしゃれさは不要。使いやすさと分かりやすさを最優先する。
**デザイン方針**:
- 見た目の装飾は最小限。影・グラデーション・アニメーションは原則使わない
- 情報が一目で把握できるレイアウト。余白とラベルで視認性を確保する
- 操作の結果が予測しやすいUI。ボタンには何が起きるかを明示するラベルをつける
- 現在の状態（処理中・完了・エラー等）が常にユーザーに伝わること

---

## 画面遷移

```
画面A（アップロード） → 画面B（バッチ履歴 + 処理状況 + 確認・編集） → 画面C（データ一覧 + エクスポート）
```

- 画面A → 画面B: アップロード完了後に自動遷移（`/batch/{batchId}`）
- 画面B のサイドバー: 過去のバッチを選択して詳細を表示可能
- 画面B → 画面C: ナビゲーションで遷移（常にアクセス可能）
- 画面A・B・Cはヘッダーのナビゲーションで相互に行き来できる
- ヘッダーの「処理状況」リンクは `/batch`（バッチ未選択状態）に遷移

---

## 共通レイアウト

### ヘッダー
- 左: アプリ名「領収書 OCR」
- 中央〜右: ナビゲーションリンク 3つ
  - 「アップロード」（画面A: `/upload`）
  - 「処理状況」（画面B: `/batch`）
  - 「データ一覧」（画面C: `/receipts`）
- 現在表示中の画面のリンクはアクティブ状態（下線やハイライト）で示す

### レイアウト
- 最大幅: 1280px、左右中央揃え
- レスポンシブ: 最低限対応（モバイル専用UIは不要）

## 画面A: アップロード画面

### 目的
領収書の画像ファイル（JPEG / PNG / PDF）をアップロードする。

### レイアウト構成

#### メインエリア（画面中央）

**ドラッグ＆ドロップゾーン**
- 破線の矩形エリア（装飾は不要、枠線のみ）
- 中央にテキスト:
  - 「ファイルをドラッグ＆ドロップしてください」
  - 「または」
  - 「ファイルを選択」ボタン
- ファイルホバー時（ドラッグ中）は枠線の色が変わる程度でよい

**アップロード制約の表示**（ドロップゾーンの下に小さく）
- 対応形式: JPEG, PNG, PDF
- 画像は自動圧縮されます（元ファイル上限: 20MB）
- 最大枚数: 10枚

#### ファイルリスト（ファイル選択後に表示）

ドロップゾーンの下にリスト形式で表示:

| 表示内容 | 説明 |
|---|---|
| ファイル名 | 元のファイル名 |
| ファイルサイズ | 例: 2.1 MB |
| ファイル形式アイコン | JPEG/PNG/PDFを区別するアイコンまたはバッジ |
| 削除ボタン | ×ボタンで個別にリストから除去 |

- ファイルが1つ以上選択されたら「アップロード開始」ボタンが有効になる
- 10枚を超えた場合はエラーメッセージを表示して追加を拒否

#### アップロード開始ボタン

- ファイルリストの下に配置
- プライマリカラーの目立つボタン
- テキスト: 「アップロード開始」（選択枚数も表示、例: 「3枚をアップロード」）
- アップロード中はボタンをローディング状態にし、二重送信を防止

### バリデーションエラー表示

| エラー条件 | 表示メッセージ |
|---|---|
| 対応していないファイル形式 | 「JPEG、PNG、PDF のみアップロードできます」 |
| ファイルサイズ超過 | 「ファイルサイズが大きすぎます（上限: 20MB）」 |
| 圧縮後サイズ超過 | 「圧縮しましたがサイズが上限（2MB）を超えています」 |
| 枚数超過 | 「一度にアップロードできるのは 10枚 までです」 |

## 画面B: 処理状況 + 確認・編集画面

### 目的
過去のアップロードバッチ一覧を表示し、選択したバッチのOCR処理状況を確認し、完了したものを確認・編集して確定する。

### レイアウト構成

画面B は2カラムレイアウトで構成する。

#### 左カラム: バッチ履歴サイドバー（固定幅 280px）

過去のアップロードバッチを新しい順に一覧表示するサイドバー。

**サイドバーヘッダー:**
- タイトル「バッチ履歴」

**バッチ一覧（各行の表示内容）:**

| 表示内容 | 説明 |
|---|---|
| 作成日時 | 例: 2026-02-28 12:00 |
| ファイル数 | 例: 3枚 |
| ステータスインジケーター | 完了/処理中/失敗の件数をバッジで表示（例: 確定済 2 / 完了 1） |

- 選択中のバッチはハイライト表示（背景色を変更）
- クリックで `navigate(/batch/{batchId})` によりメインエリアの表示を切り替え
- バッチが多い場合はサイドバー内でスクロール可能
- サイドバー下部にページネーション（「もっと読み込む」リンク）

**バッチ未選択時（`/batch` にアクセス）:**
- サイドバーは通常通り表示
- 右カラム（メインエリア）に「バッチを選択してください」のメッセージを表示

#### 右カラム: メインエリア（残りの幅を使用）

##### 上部: バッチ進捗サマリー

- プログレスバー（全体の進捗を視覚化）
  - 例: 「3/5 完了」
  - バーの色分け: 完了（緑）、処理中（青/アニメーション）、キュー待ち（グレー）、失敗（赤）
- 各ステータスの件数をバッジで表示:
  - 待機中: X件 / 処理中: X件 / 完了: X件 / 確定済: X件 / 失敗: X件

#### 中部: ジョブ一覧

各ファイルのOCRジョブをリスト形式（1行1ジョブ）で一覧表示する。各行にはステータスに応じた内容を表示する。

**各行の共通要素:**
- ファイル名
- ステータスバッジ（色付きラベル）

**ステータス別の表示内容:**

| ステータス | バッジ | 行の右側に表示する内容 |
|---|---|---|
| 待機中 | グレー「待機中」 | （なし） |
| 処理中 | 青「処理中」 | 小さなスピナー |
| 完了 | 緑「完了」 | 「確認・編集」ボタン（クリックで下部に編集フォームを展開） |
| 確定済 | 青緑「確定済」 | （なし。確定済であることがバッジで分かれば十分） |
| 失敗 | 赤「失敗」 | エラー理由テキスト + 「再試行」ボタン |

#### 下部: 確認・編集フォーム（COMPLETED のジョブを選択時に展開）

**左右2カラムレイアウト:**

**左カラム: 画像プレビュー**
- アップロードした領収書の画像をプレビュー表示（署名付きURLで取得）
- 画像のサイズは固定幅で十分（拡大・縮小は不要）

**右カラム: OCR読み取り結果の編集フォーム**

| フィールド | 入力タイプ | 必須 | 備考 |
|---|---|---|---|
| 取引日 | 日付ピッカー（YYYY-MM-DD） | ○ | OCR結果がプリセット |
| 金額（円） | 数値入力 | ○ | OCR結果がプリセット、1以上 |
| 支払先 | テキスト入力 | ○ | OCR結果がプリセット、最大200文字 |
| 品目・明細 | テキストエリア | - | OCR結果がプリセット、最大500文字 |
| 税区分 | セレクトボックスまたはテキスト | - | 「8%」「10%」等、OCR結果がプリセット |

- フォーム下部に「OCR生テキスト」の展開セクション（トグルで表示/非表示）
  - OCRが読み取った生テキスト全文を表示（読み取り専用）
  - ユーザーが生テキストを参照しながら手動修正するため

**アクションボタン:**
- 「確定」ボタン（プライマリ） — 編集内容を保存して確定
- 確定成功後、カードのステータスが CONFIRMED に更新される

### 特殊な状態

**失敗ジョブの「再試行」ボタン:**
- クリックするとジョブが再度キューに投入される（QUEUED に戻る）
- ボタン横にエラーメッセージ（例: 「OCR 処理がタイムアウトしました」）

**全ジョブ確定済み時:**
- 「すべての領収書を確定しました。データ一覧で確認・エクスポートできます。」のメッセージを表示
- 「データ一覧へ」のリンクボタン

## 画面C: データ一覧 + エクスポート画面

### 目的
確定済みのレシートデータを一覧表示し、フィルタ・ソートで絞り込み、選択したデータをCSVエクスポートする。

### レイアウト構成

#### 上部: フィルター・検索バー

横並びのフィルター入力を配置:

| フィルター項目 | 入力タイプ | 説明 |
|---|---|---|
| 期間（開始日） | 日付ピッカー | 取引日の範囲指定（開始） |
| 期間（終了日） | 日付ピッカー | 取引日の範囲指定（終了） |
| 支払先 | テキスト入力 | 店名の部分一致検索 |
| 金額（下限） | 数値入力 | 金額の範囲指定（下限） |
| 金額（上限） | 数値入力 | 金額の範囲指定（上限） |

- 「検索」ボタンでフィルタ適用
- 「クリア」ボタン（テキストリンク）でフィルタ条件をリセット

#### 中部: データテーブル

テーブル形式で確定済みレシートを一覧表示:

| 列 | 説明 | ソート可能 |
|---|---|---|
| ☐（チェックボックス） | エクスポート対象の選択用。ヘッダーに全選択チェックボックスあり | - |
| 取引日 | YYYY-MM-DD 形式 | ○ |
| 金額 | ¥X,XXX 形式（カンマ区切り） | ○ |
| 支払先 | 店名 | ○ |
| 品目 | 品目・明細（長い場合は省略表示） | - |
| 税区分 | 8% / 10% 等 | - |
| 確定日 | YYYY-MM-DD HH:mm 形式 | ○ |
| 操作 | 「編集」「削除」テキストボタン | - |

- ソート可能な列ヘッダーはクリックで昇順/降順を切り替え（▲▼アイコン付き）
- データがない場合は「確定済みのレシートデータがありません」の空状態メッセージ

#### テーブル下: ページネーション

- 「前へ」「次へ」ボタン
- ページ番号表示（例: 「1 / 3 ページ」）
- 全件数表示（例: 「全 42 件」）
- 1ページあたり20件

#### 下部固定: エクスポートアクションバー

テーブルの下、または画面下部に固定表示:

- 左: 「X件 選択中」のテキスト
- 右: 「CSVエクスポート」ボタン（プライマリ）
  - チェックが1件以上ないと無効（disabled）
  - クリック後、エクスポートジョブを作成しステータスをポーリング

**エクスポート中の表示:**
- ボタンがローディング状態に変化（スピナー + 「エクスポート中...」）
- 完了後、自動的にCSVファイルのダウンロードが開始される
- 失敗時はエラーメッセージを画面上部の通知バーで表示

### 操作列の詳細

**「編集」ボタン:**
- クリックでモーダルの編集フォームを表示
- 編集フォームの項目は画面Bと同じ（取引日、金額、支払先、品目、税区分）
- 「保存」「キャンセル」ボタン

**「削除」ボタン:**
- クリックで確認ダイアログを表示（「このレシートを削除しますか？」）
- 確認後に削除実行

## デザイン指針

### 全体の方針
- **業務ツールとしての実用性を最優先**する。見た目のおしゃれさは一切不要
- 装飾（影、グラデーション、過度なアニメーション）は使わない
- ラベル・見出し・区切り線で情報の構造を明確にする
- 操作ボタンには「何が起きるか」を明示するテキストラベルを必ずつける（アイコンのみのボタンは避ける）
- 色は「状態の区別」にのみ使い、装飾目的では使わない

### カラー
| 用途 | 色 | 使い方 |
|---|---|---|
| プライマリ操作 | 青系 | 「確定」「アップロード」等の主要アクションボタン |
| 成功・完了 | 緑 | ステータスバッジ、完了メッセージ |
| エラー・失敗 | 赤 | ステータスバッジ、エラーメッセージ、削除ボタン |
| 警告 | 黄/オレンジ | バリデーションエラー |
| 背景 | 白 | メインコンテンツ領域 |
| ヘッダー/フィルターバー | 薄いグレー | セクションの区切り |
| テキスト | 黒〜ダークグレー | 本文・ラベル |
| 補助テキスト | グレー | 説明文・ヒント |

### ステータスバッジ
| ステータス | 背景色 | テキスト |
|---|---|---|
| 待機中 | 薄いグレー | 待機中 |
| 処理中 | 薄い青 | 処理中 |
| 完了 | 薄い緑 | 完了 |
| 確定済 | 薄い青緑 | 確定済 |
| 失敗 | 薄い赤 | 失敗 |

### UI コンポーネント
- **ボタン**: 四角に近い形状（角丸は最小限）。プライマリは青の塗りつぶし+白文字、セカンダリはグレー枠線。テキストラベル必須
- **テーブル**: 罫線あり。ヘッダー行はグレー背景。ストライプ行（偶数行に薄い背景色）
- **フォーム**: ラベルはフィールドの上に配置。必須項目にはラベル横に「*」表示。入力欄はボーダーのみ（影なし）
- **エラーメッセージ**: フィールドの直下に赤文字で表示
- **確認ダイアログ**: ブラウザ標準の confirm に近いシンプルなモーダル。背景暗転+白い矩形
- **通知**: 画面上部にバー形式で表示（成功=緑背景、エラー=赤背景）。数秒で自動消去

## 補足: 各画面とAPI の対応

### 画面A
- アップロード実行: `POST /api/upload-batches`（multipart/form-data）

### 画面B
- バッチ一覧取得: `GET /api/upload-batches`（サイドバー表示用）
- ステータスポーリング: `GET /api/upload-batches/{batchId}`（2秒間隔）
- OCR結果取得: `GET /api/ocr-jobs/{jobId}`
- 画像プレビュー取得: `GET /api/ocr-jobs/{jobId}/image-url`
- 確定: `POST /api/ocr-jobs/{jobId}/confirm`
- 手動リトライ: `POST /api/ocr-jobs/{jobId}/retry`

### 画面C
- 一覧取得: `GET /api/receipts`（ページング・ソート・フィルタ対応）
- 編集: `PUT /api/receipts/{receiptId}`
- 削除: `DELETE /api/receipts/{receiptId}`
- エクスポート開始: `POST /api/exports`
- エクスポート状況ポーリング: `GET /api/exports/{exportId}`（2秒間隔）
- CSVダウンロード: `GET /api/exports/{exportId}/download`
