# シーケンス図

## 1. 概要

領収書 OCR アプリケーションの主要フローにおける処理順序を、レイヤーをまたいだシーケンス図で定義する。

### 対象フロー

| # | フロー | 概要 |
|---|---|---|
| 1 | アップロード → OCR 処理 | ファイルアップロード → MinIO 保存 → SQS 投入 → ワーカー OCR 実行 → ポーリングで完了検知 |
| 2 | 確認・確定 | OCR 結果の取得 → 画像プレビュー表示 → ユーザー編集 → 確定 → レシート作成 |
| 3 | エクスポート | レシート選択 → エクスポートジョブ作成 → ワーカー CSV 生成 → ダウンロード |

## 2. フロー①: アップロード → OCR 処理

### 2.1 全体シーケンス

ユーザーがファイルをアップロードしてから、OCR 処理が完了するまでの一連の流れ。

```mermaid
sequenceDiagram
    actor User as ユーザー
    participant FE as フロントエンド<br>(UploadPage)
    participant useUpload as useUpload
    participant uploadSvc as uploadService
    participant Ctrl as UploadBatch<br>Controller
    participant Svc as UploadBatch<br>Service
    participant OcrSvc as OcrJob<br>Service
    participant DB as PostgreSQL
    participant S3 as MinIO/S3
    participant SQS as SQS Queue
    participant Listener as OcrJob<br>MessageListener
    participant OcrWorker as OcrJob<br>Service（ワーカー）
    participant OCR as Tesseract<br>OcrEngine

    %% === アップロード ===
    User->>FE: ファイルをドラッグ＆ドロップ
    FE->>FE: useFileSelection でバリデーション<br>(形式・サイズ・枚数)
    User->>FE: 「アップロード」ボタン押下

    FE->>useUpload: upload(files)
    useUpload->>uploadSvc: createBatch(files)
    uploadSvc->>Ctrl: POST /api/upload-batches<br>(multipart/form-data)
    Ctrl->>Svc: createBatch(files, userId)

    loop ファイルごと
        Svc->>S3: putObject(storageKey, file)
        S3-->>Svc: 保存完了
    end

    Svc->>DB: INSERT upload_batches
    DB-->>Svc: batchId

    loop ファイルごと
        Svc->>OcrSvc: createJob(batchId, fileInfo)
        OcrSvc->>DB: INSERT ocr_jobs (status=QUEUED)
        DB-->>OcrSvc: jobId
        OcrSvc->>SQS: sendMessage(OcrJobMessage)
        SQS-->>OcrSvc: messageId
        OcrSvc-->>Svc: jobId
    end

    Svc-->>Ctrl: CreateBatchResponse
    Ctrl-->>uploadSvc: 201 Created
    uploadSvc-->>useUpload: response
    useUpload->>FE: navigate(/batch/{batchId})

    %% === ポーリング ===
    Note over FE,DB: 画面B に遷移後、ポーリング開始

    FE->>FE: useBatchStatus(batchId) 開始

    loop 2秒ごと（全ジョブ終端まで）
        FE->>Ctrl: GET /api/upload-batches/{batchId}
        Ctrl->>Svc: getBatchStatus(batchId)
        Svc->>DB: SELECT ocr_jobs WHERE batch_id = ?
        DB-->>Svc: ジョブ一覧 + ステータス
        Svc-->>Ctrl: BatchStatusResponse（summary 含む）
        Ctrl-->>FE: 200 OK
        FE->>FE: プログレスバー・ジョブ一覧を更新
    end

    %% === OCR ワーカー（非同期） ===
    Note over SQS,OCR: バックグラウンドで並行実行（最大3並列）

    SQS->>Listener: @SqsListener メッセージ受信
    Listener->>OcrWorker: processOcrJob(message)
    OcrWorker->>DB: UPDATE status → PROCESSING
    OcrWorker->>S3: getObject(storageKey)
    S3-->>OcrWorker: 画像データ

    alt PDF の場合
        OcrWorker->>OcrWorker: PdfConverter で画像変換
        OcrWorker->>S3: putObject(convertedImageKey)
    end

    OcrWorker->>OCR: recognize(image)

    alt OCR 成功
        OCR-->>OcrWorker: OcrResult（5項目 + 生テキスト）
        OcrWorker->>DB: UPDATE ocr_jobs SET ocr_* = ?, status = COMPLETED
    else OCR 失敗 / タイムアウト（60秒）
        OcrWorker->>DB: UPDATE status → FAILED, retry_count++
        OcrWorker--xSQS: 例外スロー（メッセージ未削除 → 自動再配信）
    end
```

### 2.2 失敗→自動リトライのシーケンス

```mermaid
sequenceDiagram
    participant SQS as SQS Queue
    participant DLQ as DLQ
    participant Listener as OcrJob<br>MessageListener
    participant Worker as OcrJob<br>Service
    participant DB as PostgreSQL

    Note over SQS,DB: 初回処理失敗
    SQS->>Listener: メッセージ受信（受信回数 = 1）
    Listener->>Worker: processOcrJob(message)
    Worker->>DB: UPDATE status → PROCESSING
    Worker->>Worker: OCR 処理実行
    Worker->>DB: UPDATE status → FAILED, retry_count = 1
    Worker--xSQS: 例外スロー

    Note over SQS,DB: 120秒後に再配信（リトライ1回目）
    SQS->>Listener: メッセージ再配信（受信回数 = 2）
    Listener->>Worker: processOcrJob(message)
    Worker->>DB: ステータス確認（冪等性チェック）
    Worker->>DB: UPDATE status → PROCESSING
    Worker->>Worker: OCR 処理実行
    Worker->>DB: UPDATE status → FAILED, retry_count = 2
    Worker--xSQS: 例外スロー

    Note over SQS,DB: 120秒後に再配信（リトライ2回目）
    SQS->>Listener: メッセージ再配信（受信回数 = 3）
    Listener->>Worker: processOcrJob(message)
    Worker->>DB: UPDATE status → PROCESSING
    Worker->>Worker: OCR 処理実行
    Worker->>DB: UPDATE status → FAILED, retry_count = 3
    Worker--xSQS: 例外スロー

    Note over SQS,DLQ: maxReceiveCount=3 に到達 → DLQ に移動
    SQS->>DLQ: メッセージ移動
```

### 2.3 失敗→手動リトライのシーケンス

```mermaid
sequenceDiagram
    actor User as ユーザー
    participant FE as フロントエンド<br>(OcrJobPage)
    participant useRetry as useRetryJob
    participant ocrSvc as ocrJobService
    participant Ctrl as OcrJob<br>Controller
    participant Svc as OcrJob<br>Service
    participant DB as PostgreSQL
    participant SQS as SQS Queue

    Note over User,SQS: 自動リトライ3回失敗後

    FE->>FE: JobRow に「再試行」ボタン表示（status=FAILED）
    User->>FE: 「再試行」ボタン押下
    FE->>useRetry: retryJob(jobId)
    useRetry->>ocrSvc: retryJob(jobId)
    ocrSvc->>Ctrl: POST /api/ocr-jobs/{jobId}/retry
    Ctrl->>Svc: retryJob(jobId, userId)
    Svc->>DB: SELECT ocr_jobs WHERE id = ? AND user_id = ?
    DB-->>Svc: OcrJob（status=FAILED）
    Svc->>DB: UPDATE retry_count = 0, status = QUEUED
    Svc->>SQS: sendMessage(OcrJobMessage)
    SQS-->>Svc: messageId
    Svc-->>Ctrl: OcrJobResponse
    Ctrl-->>ocrSvc: 200 OK
    ocrSvc-->>useRetry: response
    useRetry->>FE: ポーリング再開（useBatchStatus）
```

## 3. フロー②: 確認・確定

ユーザーが OCR 完了済みジョブの結果を確認・編集して確定するフロー。

```mermaid
sequenceDiagram
    actor User as ユーザー
    participant FE as フロントエンド<br>(OcrJobPage)
    participant useOcrJob as useOcrJob
    participant ocrSvc as ocrJobService
    participant Ctrl as OcrJob<br>Controller
    participant Svc as OcrJob<br>Service
    participant RcptSvc as Receipt<br>Service
    participant DB as PostgreSQL
    participant S3 as MinIO/S3

    Note over User,S3: ジョブ status=COMPLETED 後、ユーザーが JobRow をクリック

    %% === OCR 結果取得 ===
    User->>FE: COMPLETED のジョブを選択
    FE->>useOcrJob: fetchJob(jobId)

    par OCR 結果と画像 URL を並行取得
        useOcrJob->>ocrSvc: fetchOcrJob(jobId)
        ocrSvc->>Ctrl: GET /api/ocr-jobs/{jobId}
        Ctrl->>Svc: getJob(jobId, userId)
        Svc->>DB: SELECT ocr_jobs WHERE id = ?
        DB-->>Svc: OcrJob（ocrResult 含む）
        Svc-->>Ctrl: OcrJobResponse
        Ctrl-->>ocrSvc: 200 OK
        ocrSvc-->>useOcrJob: OcrJob データ
    and
        useOcrJob->>ocrSvc: fetchImageUrl(jobId)
        ocrSvc->>Ctrl: GET /api/ocr-jobs/{jobId}/image-url
        Ctrl->>Svc: getImageUrl(jobId, userId)
        Svc->>S3: generatePresignedUrl(storageKey)
        Note over Svc,S3: PDF の場合は<br>convertedImageKey を使用
        S3-->>Svc: 署名付き URL（有効期限15分）
        Svc-->>Ctrl: ImageUrlResponse
        Ctrl-->>ocrSvc: 200 OK
        ocrSvc-->>useOcrJob: imageUrl
    end

    useOcrJob-->>FE: OcrJob + imageUrl

    %% === 確認・編集画面の表示 ===
    FE->>FE: EditSection 表示
    FE->>FE: ImagePreview に画像表示
    FE->>FE: ConfirmForm に OCR 結果をセット<br>（date, amount, storeName, description, taxCategory）
    FE->>FE: OcrRawText に生テキスト表示

    %% === ユーザー編集・確定 ===
    User->>FE: フォーム内容を修正
    User->>FE: 「確定」ボタン押下

    FE->>FE: useConfirmJob: フォームバリデーション<br>（date: 必須, amount: 必須・1以上, storeName: 必須・200文字以内）

    alt バリデーションエラー
        FE->>FE: エラーメッセージ表示
    else バリデーション OK
        FE->>ocrSvc: confirmJob(jobId, confirmData)
        ocrSvc->>Ctrl: POST /api/ocr-jobs/{jobId}/confirm
        Ctrl->>Ctrl: @Valid でリクエストバリデーション
        Ctrl->>Svc: confirmJob(jobId, confirmData, userId)

        Svc->>DB: SELECT ocr_jobs WHERE id = ? AND user_id = ?
        DB-->>Svc: OcrJob

        alt ステータスが COMPLETED でない
            Svc-->>Ctrl: DomainException(JOB_NOT_RETRYABLE)
            Ctrl-->>ocrSvc: 409 Conflict
            ocrSvc-->>FE: エラー
            FE->>FE: Notification でエラー表示
        else ステータスが COMPLETED
            Svc->>DB: UPDATE ocr_jobs SET status = CONFIRMED

            Svc->>RcptSvc: createReceipt(jobId, confirmData, userId)
            RcptSvc->>DB: INSERT receipts
            DB-->>RcptSvc: receiptId
            RcptSvc-->>Svc: Receipt

            Svc-->>Ctrl: ReceiptResponse
            Ctrl-->>ocrSvc: 201 Created
            ocrSvc-->>FE: response
            FE->>FE: Notification で成功表示
            FE->>FE: ジョブ一覧を再取得（ステータス更新）
        end
    end
```

## 4. フロー③: エクスポート

確定済みレシートを選択して CSV エクスポートするフロー。

### 4.1 全体シーケンス

```mermaid
sequenceDiagram
    actor User as ユーザー
    participant FE as フロントエンド<br>(ReceiptListPage)
    participant useReceipts as useReceipts
    participant useExport as useExport
    participant rcptSvc as receiptService
    participant expSvc as exportService
    participant RcptCtrl as Receipt<br>Controller
    participant ExpCtrl as Export<br>Controller
    participant RcptService as Receipt<br>Service
    participant ExpService as Export<br>Service
    participant CsvGen as CsvGenerator
    participant DB as PostgreSQL
    participant SQS as SQS Queue
    participant Listener as ExportJob<br>MessageListener
    participant S3 as MinIO/S3

    %% === 一覧表示 ===
    FE->>useReceipts: fetchReceipts(params)
    useReceipts->>rcptSvc: fetchReceipts({page, size, sort, filters})
    rcptSvc->>RcptCtrl: GET /api/receipts?page=0&size=20&sort=date&order=desc
    RcptCtrl->>RcptService: searchReceipts(searchRequest, userId)
    RcptService->>DB: SELECT receipts WHERE user_id = ?<br>（フィルタ・ソート・ページング）
    DB-->>RcptService: レシート一覧 + 総件数
    RcptService-->>RcptCtrl: PageResponse<ReceiptResponse>
    RcptCtrl-->>rcptSvc: 200 OK
    rcptSvc-->>useReceipts: response
    useReceipts-->>FE: レシート一覧表示

    %% === フィルタ・ソート ===
    User->>FE: FilterBar でフィルタ条件を入力
    User->>FE: 「検索」ボタン押下
    FE->>useReceipts: fetchReceipts(updatedParams)
    Note over FE,DB: 上記と同じ GET /api/receipts（パラメータ変更）

    %% === 対象選択 ===
    User->>FE: チェックボックスでレシート選択
    FE->>FE: useReceiptSelection で選択状態管理

    %% === エクスポート実行 ===
    User->>FE: 「CSV エクスポート」ボタン押下
    FE->>useExport: startExport(selectedReceiptIds)
    useExport->>expSvc: createExport(receiptIds)
    expSvc->>ExpCtrl: POST /api/exports
    ExpCtrl->>ExpService: createExport(receiptIds, userId)
    ExpService->>DB: INSERT export_jobs (status=QUEUED)
    DB-->>ExpService: exportId
    ExpService->>DB: INSERT export_job_receipts（中間テーブル）
    ExpService->>SQS: sendMessage(ExportJobMessage)
    SQS-->>ExpService: messageId
    ExpService-->>ExpCtrl: ExportJobResponse
    ExpCtrl-->>expSvc: 201 Created
    expSvc-->>useExport: exportId

    %% === ポーリング ===
    useExport->>useExport: usePolling 開始（2秒間隔）

    loop 2秒ごと（COMPLETED または FAILED まで）
        useExport->>expSvc: fetchExportStatus(exportId)
        expSvc->>ExpCtrl: GET /api/exports/{exportId}
        ExpCtrl->>ExpService: getExportStatus(exportId, userId)
        ExpService->>DB: SELECT export_jobs WHERE id = ?
        DB-->>ExpService: ExportJob
        ExpService-->>ExpCtrl: ExportJobResponse
        ExpCtrl-->>expSvc: 200 OK
        expSvc-->>useExport: status
    end

    %% === ワーカー（非同期） ===
    Note over SQS,S3: バックグラウンドで処理

    SQS->>Listener: @SqsListener メッセージ受信
    Listener->>ExpService: processExport(message)
    ExpService->>DB: UPDATE status → PROCESSING
    ExpService->>DB: SELECT receipts<br>JOIN export_job_receipts<br>WHERE export_job_id = ?
    DB-->>ExpService: レシートデータ一覧
    ExpService->>CsvGen: generate(receipts)
    CsvGen-->>ExpService: CSV バイト列
    ExpService->>S3: putObject(storageKey, csv)
    S3-->>ExpService: 保存完了
    ExpService->>DB: UPDATE status → COMPLETED,<br>file_name, storage_key を記録

    %% === ダウンロード ===
    Note over FE,S3: ポーリングで COMPLETED を検知
    useExport->>useExport: ポーリング停止
    useExport->>expSvc: downloadExport(exportId)
    expSvc->>ExpCtrl: GET /api/exports/{exportId}/download
    ExpCtrl->>ExpService: downloadExport(exportId, userId)
    ExpService->>DB: SELECT export_jobs WHERE id = ?
    alt ステータスが COMPLETED でない
        ExpService-->>ExpCtrl: DomainException(EXPORT_NOT_READY)
        ExpCtrl-->>expSvc: 409 Conflict
    else ステータスが COMPLETED
        ExpService->>S3: getObject(storageKey)
        S3-->>ExpService: CSV データ
        ExpService-->>ExpCtrl: CSV データ + ファイル名
        ExpCtrl-->>expSvc: 200 OK<br>(Content-Type: text/csv)
        expSvc-->>FE: Blob データ
        FE->>FE: ブラウザダウンロード開始
        FE->>FE: Notification で成功表示
    end
```

### 4.2 レシート編集・削除のシーケンス

```mermaid
sequenceDiagram
    actor User as ユーザー
    participant FE as フロントエンド<br>(ReceiptListPage)
    participant rcptSvc as receiptService
    participant Ctrl as Receipt<br>Controller
    participant Svc as Receipt<br>Service
    participant DB as PostgreSQL

    %% === レシート編集 ===
    User->>FE: 一覧の「編集」ボタン押下
    FE->>FE: EditReceiptModal を表示
    FE->>rcptSvc: fetchReceipt(receiptId)
    rcptSvc->>Ctrl: GET /api/receipts/{receiptId}
    Ctrl->>Svc: getReceipt(receiptId, userId)
    Svc->>DB: SELECT receipts WHERE id = ? AND user_id = ?
    DB-->>Svc: Receipt
    Svc-->>Ctrl: ReceiptResponse
    Ctrl-->>rcptSvc: 200 OK
    rcptSvc-->>FE: レシート詳細

    FE->>FE: フォームに現在値をセット
    User->>FE: 内容を修正して「保存」押下

    FE->>rcptSvc: updateReceipt(receiptId, data)
    rcptSvc->>Ctrl: PUT /api/receipts/{receiptId}
    Ctrl->>Ctrl: @Valid でリクエストバリデーション
    Ctrl->>Svc: updateReceipt(receiptId, data, userId)
    Svc->>DB: UPDATE receipts SET ... WHERE id = ? AND user_id = ?
    DB-->>Svc: 更新結果
    Svc-->>Ctrl: ReceiptResponse
    Ctrl-->>rcptSvc: 200 OK
    rcptSvc-->>FE: response
    FE->>FE: モーダルを閉じる
    FE->>FE: 一覧を再取得
    FE->>FE: Notification で成功表示

    %% === レシート削除 ===
    User->>FE: 一覧の「削除」ボタン押下
    FE->>FE: DeleteConfirmDialog を表示
    User->>FE: 「はい」を選択

    FE->>rcptSvc: deleteReceipt(receiptId)
    rcptSvc->>Ctrl: DELETE /api/receipts/{receiptId}
    Ctrl->>Svc: deleteReceipt(receiptId, userId)
    Svc->>DB: DELETE receipts WHERE id = ? AND user_id = ?
    Note over Svc,DB: export_job_receipts は<br>ON DELETE CASCADE で自動削除
    DB-->>Svc: 削除完了
    Svc-->>Ctrl: void
    Ctrl-->>rcptSvc: 204 No Content
    rcptSvc-->>FE: response
    FE->>FE: 一覧を再取得
    FE->>FE: Notification で成功表示
```

## 5. クラス間の呼び出し関係まとめ

各フローで登場するクラスの呼び出し関係を整理する。

### 5.1 フロー①: アップロード → OCR

```mermaid
graph LR
    subgraph Presentation
        UBC[UploadBatchController]
    end

    subgraph Application
        UBS[UploadBatchService]
        OJS[OcrJobService]
        OJML["OcrJobMessageListener<br/>(@SqsListener)"]
    end

    subgraph Infrastructure
        MSC[MinioStorageClient]
        UBRI[UploadBatchRepositoryImpl]
        OJRI[OcrJobRepositoryImpl]
        SMS[SqsMessageSender]
        TOE[TesseractOcrEngine]
        PDF[PdfConverter]
    end

    UBC --> UBS
    UBS -->|MinIO保存| MSC
    UBS -->|DB| UBRI
    UBS --> OJS
    OJS -->|DB| OJRI
    OJS -->|キュー投入| SMS
    SMS -.->|非同期| OJML
    OJML --> OJS
    OJS -->|画像取得| MSC
    OJS -->|OCR実行| TOE
    OJS -->|PDF変換| PDF
```
